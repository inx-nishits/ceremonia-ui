## Implementation Notes

### Field Types:
- **Text input**: Short answers (names, dates, single-line responses)
- **Textarea**: Long-form answers (stories, descriptions, anecdotes)
- **Number input**: Phone numbers, guest count
- **Date input**: Ceremony dates
- **Dropdown/Select**: Yes/No questions, predefined choices (rituals, music timing, tone selection)
- **Dynamic fields**: Add multiple entries (bridal party members, witnesses, speakers, music selections)
- **Checkbox**: Multiple selection options (ceremony tone, exit options)
- **Radio buttons**: Single selection from multiple options (narrative style)

### Special Features:
- **Block 1**: Locked after first submission (non-editable)
- **Blocks 2-15**: Always editable
- **Block 5**: Dynamic "Add person" button for bridal party members
- **Block 7**: Dynamic "Add witness" and "Add speaker" buttons
- **Block 12**: Structured music selection with song title + artist fields
- **Block 13**: AI-generated content based on speaker information from Block 7
- **Block 15**: Multiple selection for tone, single selection for narrative style

### Validation Rules:
- Phone numbers: Numeric only
- Dates: Valid date format
- Required fields: All questions marked as required must be answered before proceeding
- Character limits: Apply appropriate maxlength to prevent excessive input
- Email validation: If email fields are added

### Dynamic Variable Replacement:
Throughout the questionnaire, replace generic terms with actual names:
- "Partner 1" → Use the first name entered in Block 1, Question 1
- "Partner 2" → Use the first name entered in Block 1, Question 4
- "[Speaker Name]" → Use actual speaker names from Block 7
